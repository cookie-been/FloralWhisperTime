# 运维 SOP 手册

本文档面向运维、实施和售后人员，按日常、每周、升级前和故障时四类场景整理标准操作流程。

## 1. 日常 SOP

建议每日执行一次。

### 1.1 日常检查流程

1. 打开首页，确认站点可访问
2. 打开后台并确认可登录
3. 打开运维中心查看系统状态
4. 查看是否存在未处理风险提示
5. 查看未读留言数量是否异常增长

### 1.2 日常关注项

重点关注：

- 数据库是否正常
- 上传目录是否可写
- 磁盘空间是否充足
- 是否有最近备份
- AI 状态是否正常
- 限流 / 繁忙拒绝次数是否异常增长

## 2. 每周 SOP

建议每周至少执行一次。

### 2.1 每周维护流程

1. 执行一次完整备份
2. 在运维中心执行一次巡检
3. 导出一份当前配置包
4. 检查操作日志是否存在异常写操作
5. 检查磁盘和上传目录容量趋势

## 3. 升级前 SOP

### 3.1 升级前准备

1. 确认本次升级版本说明
2. 确认 `.env` 是否需要新增或修改配置
3. 确认业务低峰窗口
4. 确认最近一次备份可用

### 3.2 升级前执行项

1. 执行：

```bash
./backup.sh
```

2. 记录当前版本与提交号
3. 确认当前后台和首页可正常访问
4. 确认升级回滚入口已知

### 3.3 升级执行

源码模式：

```bash
./upgrade.sh
```

release 模式：

```bash
./release-upgrade.sh
```

### 3.4 升级后复核

1. 首页可访问
2. `/api/health` 正常
3. 后台可登录
4. 作品管理正常
5. 站点配置正常
6. 运维中心正常
7. 如启用 AI，完成一次 AI 调用验证

## 4. 故障处理 SOP

### 4.1 站点打不开

按顺序检查：

1. `docker compose ps`
2. `docker compose logs --tail=200 web`
3. `docker compose logs --tail=200 backend`
4. `docker compose logs --tail=200 mysql`
5. `/api/health`

### 4.2 后台登录异常

按顺序检查：

1. 是否缓存旧 token
2. `ADMIN_USERNAME`
3. `ADMIN_PASSWORD`
4. 数据库中的管理员安全状态
5. 后端日志

### 4.3 图片上传异常

按顺序检查：

1. 上传目录是否存在
2. 上传目录是否可写
3. 磁盘空间是否足够
4. 图片是否超过 20MB
5. 是否触发上传并发保护

### 4.4 AI 生图异常

按顺序检查：

1. AI 是否启用
2. AI Key 是否配置
3. 模型名是否正确
4. 接口地址是否正确
5. 是否触发 AI 并发保护

### 4.5 数据异常或误操作

按顺序处理：

1. 查操作日志
2. 能按日志恢复则优先恢复
3. 能按逻辑删除恢复则走恢复
4. 必要时再做配置导入或备份恢复

## 5. 备份 SOP

### 5.1 手动备份

```bash
./backup.sh
```

或在后台运维中心创建备份任务。

### 5.2 备份后确认

确认：

- 备份目录已生成
- 运维中心能看到最近备份
- 最近备份可下载

## 6. 回滚 SOP

### 6.1 回滚前

1. 明确问题是否必须回滚
2. 先做 `--dry-run`
3. 确认目标备份可用

### 6.2 回滚执行

源码模式：

```bash
./rollback.sh --latest --dry-run
./rollback.sh --latest
```

release 模式：

```bash
./release-rollback.sh --latest-previous
```

### 6.3 回滚后

1. 检查首页
2. 检查后台登录
3. 检查运维中心
4. 检查核心业务页面

## 7. 留档 SOP

建议每次重大操作后都留档：

- 当前时间
- 当前版本
- 当前提交号
- 操作人
- 操作内容
- 验证结果

适合留档的场景：

- 首次部署
- 正式升级
- 回滚
- 重大配置调整
- 故障恢复
