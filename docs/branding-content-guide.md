# 内容填写规范

本文档用于指导客户、运营和实施人员填写首页文案、品牌故事、关于我们、作品描述和 AI 提示词，尽量让系统整体内容更统一、更专业。

## 1. 总原则

建议所有内容遵循：

- 简洁
- 真实
- 统一风格
- 少空话
- 少堆砌词藻
- 多写具体场景和服务价值

## 2. 首页文案怎么写

首页核心通常包括：

- 眉题
- 主标题
- 简介
- 按钮文案

### 2.1 眉题

建议：

- 4 到 10 个字
- 用来概括品牌定位或服务特征

示例：

- 城市花艺工作室
- 婚礼与仪式花艺
- 日常与节日花礼

### 2.2 主标题

建议：

- 一句话表达品牌主张
- 不要太长
- 尽量读起来顺

示例：

- 用花表达爱与生活
- 为重要时刻准备更有温度的花艺
- 从日常花礼到婚礼现场的完整花艺表达

### 2.3 首页简介

建议：

- 1 到 3 句话
- 讲清楚服务场景、风格和价值

示例：

```text
专注婚礼、生日、开业和节日花艺设计，
以自然、克制、耐看的表达方式，
为日常与重要时刻提供更稳定的花艺服务。
```

### 2.4 按钮文案

建议用明确动作，不要太虚：

- 查看作品
- 立即咨询
- 了解我们
- 联系门店

## 3. 品牌故事怎么写

品牌故事建议回答这几个问题：

1. 你是谁
2. 为什么做这件事
3. 你擅长什么
4. 你希望带给客户什么感受

建议结构：

- 第一段：品牌起点
- 第二段：擅长方向
- 第三段：服务理念

不建议：

- 全是空泛形容词
- 大段自我感动式表达
- 只写“热爱鲜花、热爱生活”但没有具体内容

## 4. About 页内容怎么写

About 页通常包括：

- 标题 / 副标题
- 品牌故事正文
- 时间轴
- 团队成员

### 4.1 标题和副标题

建议：

- 标题直接表达主题
- 副标题补充服务和气质

示例：

- 标题：关于我们
- 副标题：用花表达空间、情绪与重要时刻

### 4.2 时间轴

建议：

- 一条只写一件事
- 按年份或阶段清晰排列
- 重点写品牌成长节点

示例：

- 2021：工作室成立，开始服务本地鲜花订制客户
- 2023：拓展婚礼与活动花艺服务
- 2025：完善品牌展示系统与线上内容运营

### 4.3 团队成员

建议：

- 姓名
- 职责
- 一句简短介绍

示例：

```text
主理人 / 花艺设计师
负责整体花艺风格把控与重点客户方案设计。
```

## 5. 作品描述怎么写

作品内容通常包括：

- 名称
- 描述
- 花材
- 标签
- 寓意

### 5.1 名称

建议：

- 简短
- 容易记
- 有画面感

示例：

- 晨雾誓约
- 白昼来信
- 森屿序曲

### 5.2 描述

建议：

- 写风格、配色、适用场景
- 避免只写“很好看”“很高级”

示例：

```text
以白玫瑰、洋牡丹和轻盈绿植组成，
适合婚礼仪式、纪念日或偏克制的送礼场景。
```

### 5.3 花材

建议写主要花材，不要乱堆太多。

### 5.4 标签

建议控制在 3 到 6 个：

- 婚礼
- 白绿色
- 轻法式
- 生日
- 开业

### 5.5 寓意

建议：

- 1 句话即可
- 写情绪和祝福，不要太空

示例：

- 象征纯净承诺与温柔陪伴
- 适合表达祝福、感谢与坚定心意

## 6. 联系页文案怎么写

联系页建议写得更直接：

- 门店位置
- 联系方式
- 营业时间
- 留言引导语

留言引导语建议：

- 欢迎留下你的花礼需求、预算范围和使用场景
- 婚礼、生日、开业、节日订制都可以先留言沟通

## 7. 后台文案怎么写

后台页面文案建议偏克制、偏说明型，不要写成营销海报。

例如：

- 作品管理：筛选、整理与更新作品内容
- 站点配置：统一维护首页、门店、品牌故事与关于我们
- 运维中心：统一查看系统状态并执行备份、巡检和迁移

## 8. AI 提示词怎么写更容易出效果

建议结构：

1. 场景
2. 花束风格
3. 色彩
4. 主要花材
5. 光线 / 氛围
6. 构图要求

示例：

```text
一束现代婚礼手捧花，
白绿色配色，
以白玫瑰、洋牡丹、尤加利为主，
自然蓬松结构，
室内柔和自然光，
高级、干净、克制，
适合品牌展示海报。
```

不建议：

- 只写“两束好看的花”
- 词太少
- 全是抽象形容词，没有场景和颜色

## 9. 常见问题

### 9.1 文案太长

首页、标题、副标题不要过长，否则前台容易显得拖沓。

### 9.2 语气不统一

品牌内容、作品描述、关于我们和联系页尽量用同一种语气风格。

### 9.3 AI 提示词太空

AI 提示词越具体，生成结果通常越稳定。
