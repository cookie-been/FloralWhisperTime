# 商业交付说明

本文档用于说明项目交付边界、推荐交付物和客户侧维护要求。

## 1. 当前交付范围

本系统当前交付包含：

- PC Web 前台
- PC Web 管理后台
- 微信小程序前台源码
- Java + MySQL 后端
- Docker 化部署链路
- 备份、恢复、升级、回滚脚本

## 2. 当前不包含的能力

以下能力不在本次标准交付范围：

- 在线支付
- 订单管理
- 库存管理
- 会员系统
- 多租户 SaaS 隔离
- 自动计费/授权服务

## 3. 建议交付物

建议对客户交付以下内容：

- 源码仓库或源码包
- `.env.production.example`
- 本目录正式文档
- 品牌 Logo 与必要素材
- 默认管理员账号初始化方式说明

## 4. 推荐商业交付方式

可按以下两种方式交付：

1. 源码授权交付  
   适合具备自主部署能力的客户。

2. 托管部署交付  
   由交付方负责部署、升级、备份和故障恢复。

## 5. 客户侧最低维护要求

- 保存 `.env` 和管理员密码
- 在后台维护站点信息、关于我们内容和 AI 配置
- 定期备份数据库与上传文件
- 升级前先执行备份
- 保证 Docker 运行环境稳定
- 域名、HTTPS、服务器安全由运维方负责

## 6. 推荐交付流程

推荐按以下顺序完成正式交付：

1. 部署目标环境并完成基础验收
2. 在后台维护站点信息、关于我们内容和 AI 配置
3. 在“系统状态”页导出一份配置包留档
4. 执行一次完整备份，确认可下载最近备份
5. 将源码、文档、配置包和备份策略一并交付

说明：

- 配置包适合做客户环境初始化、同品牌多实例复制和售后恢复前基线留存
- 配置包不会包含作品数据、留言数据和操作日志
- 标准发布版数据库默认保持干净，不预置演示作品、演示团队、演示时间轴和演示留言
- 需要完整迁移业务数据时，仍应结合数据库备份和上传目录备份

## 7. 授权建议

如果用于正式商业售卖，建议至少补充以下法律/商务材料：

- 软件授权协议
- 售后服务范围说明
- 版本升级与维护周期说明
- 数据责任边界说明

当前仓库内未内置正式法律文本，建议根据你的商业模式单独制定。
