# 花语时光功能说明

## 1. 产品定位

花语时光是一个面向鲜花门店与花艺工作室的品牌展示系统，核心目标是：

- 展示品牌形象与门店信息
- 展示作品内容与分类
- 收集访客咨询与留言
- 提供后台内容运营、AI 辅助与基础运维能力

当前版本不包含交易闭环，不支持：

- 购物车
- 在线支付
- 订单管理
- 库存管理
- 会员中心

## 2. 终端范围

系统包含三个终端：

- PC Web 前台
- PC Web 管理后台
- 微信小程序前台

## 3. Web 前台功能

### 3.1 首页

首页支持以下内容展示：

- 品牌首屏文案
- 首页轮播图
- 品牌 Logo
- 真实统计数据
- 精选作品区
- 服务分类入口
- 品牌故事摘要
- 门店与联系入口

说明：

- 首页轮播图已支持后台动态配置
- 首页统计数据不再人工录入，直接按系统真实作品数据计算

### 3.2 作品画廊

作品画廊支持：

- 分类筛选
- 关键词搜索
- 排序切换
- 分页浏览
- 查看作品详情

排序方式包括：

- 精选优先
- 最新
- 价格升序
- 价格降序

### 3.3 作品详情

作品详情页展示：

- 多图画廊
- 作品名称
- 分类
- 价格
- 花材
- 寓意
- 标签
- 相关推荐

### 3.4 关于我们

关于我们页面展示：

- 页首主视觉
- 标题与副标题
- 品牌故事正文
- 时间轴
- 团队成员
- 门店相关信息

以上内容均支持后台动态维护。

### 3.5 联系我们

联系页支持展示：

- 门店地址
- 电话
- 微信
- 地图位置
- 营业时间
- 留言表单
- 联系场景配图

留言提交后会写入数据库，并进入后台“用户留言”模块。

## 4. Web 管理后台功能

### 4.1 管理员登录

管理员通过 `/admin/login` 登录后台。

当前登录与安全策略包括：

- JWT Bearer Token 鉴权
- 首次交付默认管理员可登录
- 首次登录后强制修改管理员密码
- 管理员密码修改后以 BCrypt 形式持久化

### 4.2 一级菜单

当前后台包含 7 个一级入口：

- 运营总览
- 作品管理
- 站点配置
- AI 生图配置
- 用户留言
- 运维中心
- 操作日志

补充说明：

- 后台左侧保留一级菜单
- 顶部已提供“已打开页面”导航条，便于在多个工作区间快速切换
- 当前打开页签会保存在浏览器本地存储中，刷新后仍可恢复

### 4.3 运营总览

运营总览主要展示：

- 作品总数
- 精选作品占比
- 分类分布
- 最新作品
- 未读留言数量
- 首页主视觉概览
- 后台快捷入口

### 4.4 作品管理

作品管理支持：

- 列表查看
- 搜索与筛选
- 新增作品
- 编辑作品
- 单条删除
- 批量删除
- 封面与多图上传
- 精选状态维护
- 排序维护

作品字段包括：

- 名称
- 分类
- 图片列表
- 价格
- 描述
- 花材
- 寓意
- 标签
- 是否精选
- 排序
- 创建时间

补充说明：

- 当前删除已改为逻辑删除
- 后台支持按 `有效 / 已删除 / 全部` 查看作品
- 已删除作品支持恢复

### 4.5 站点配置

站点配置是统一的动态配置入口，当前页面采用 Tabs 方式拆分管理，主要分组包括：

- 首页与品牌
- 门店与联系
- 品牌故事
- 关于我们
- 后台文案
- 媒体资源

维护内容包括：

- 首页文案与按钮
- 首页轮播图
- 品牌 Logo
- 门店电话、微信、地址、坐标、营业时间
- 品牌故事标题、正文与配图
- About 页首图、标题、副标题、正文
- About 时间轴
- About 团队成员
- 后台各一级页面标题、副标题、说明文案
- 联系页配图
- 管理员登录页轮播图

补充说明：

- 媒体资源已支持多图上传
- 首页轮播、登录页轮播、联系页配图、品牌故事配图均可在后台持续维护
- About 内容已和站点配置统一到同一入口维护
- 后台各一级页面标题、副标题、说明文案支持动态配置
- 带“小程序同步”标识的资源会同步影响小程序对应页面

### 4.6 AI 生图配置

AI 生图配置为独立后台页面，支持维护：

- AI 启用状态
- 提供商
- API Key
- 图片模型
- 文本模型
- 生成地址与路径
- 图片尺寸
- 文本温度与最大输出

当前工作流为：

1. 在 AI 配置中维护密钥和模型
2. 在作品管理中输入提示词
3. 可上传最多 3 张参考图
4. 生成 1 张图片并下载到本地 `uploads/ai/`
5. 人工确认后进入作品新增流程
6. 可继续调用 AI 作品信息建议补全名称、花材、标签、寓意等内容

### 4.7 用户留言

用户留言模块支持：

- 留言列表查看
- 关键字搜索
- 已读 / 未读筛选
- 标记已读
- 删除与恢复

补充说明：

- 留言数据存储在数据库中
- 当前删除已改为逻辑删除

### 4.8 运维中心

运维中心当前已按 Tabs 拆分，主要包含：

- 总览
- 备份与下载
- 巡检与任务
- 安全与风险
- 日志归档
- 配置迁移

支持查看或执行：

- 当前部署版本、提交号、部署时间
- 数据库状态与容量
- 上传目录状态与文件量
- 磁盘状态
- AI 配置状态
- 最近备份
- 手动备份
- 系统巡检
- 配置导出与导入
- 操作日志归档
- 最近运维任务记录
- 基础限流与并发保护状态
- 统一动作卡片
- 动作对应命令参考
- 最近一次动作结果摘要
- 动作执行前确认提示
- 本次归档文件快捷下载
- 动作执行人和来源展示
- 部署、升级、回滚边界只读说明

补充说明：

- 风险项支持直接定位到相关处理区域
- 任务页支持查看结构化结果和命令参考
- 运维中心用于低风险、低频但需要留痕的后台运维动作
- 部署、升级、回滚仍建议通过 Linux / Docker 命令行脚本执行

### 4.9 操作日志

操作日志用于审计后台写操作与恢复误操作数据，支持：

- 后台写操作记录
- 登录行为记录
- 关键筛选
- 详情查看
- CSV 导出
- 快照恢复
- 恢复链路追踪

补充说明：

- 当前提供快捷视图：全部日志、仅失败、仅可恢复、恢复记录
- 当前提供常用预设：今日失败、今日恢复、作品变更、留言处理、AI 配置
- 日志详情中可查看恢复链路、失败原因、前后快照和结构化差异
- 查询类请求默认不记操作日志
- 恢复动作本身也会被记录

## 5. 数据管理与恢复策略

### 5.1 逻辑删除

以下模块当前已使用逻辑删除：

- 作品
- 用户留言
- About 时间轴
- About 团队成员

逻辑删除的价值：

- 避免误删直接丢失数据
- 可在后台恢复
- 便于结合操作日志追溯

### 5.2 操作日志恢复

支持按日志快照恢复的场景包括：

- 作品内容误改
- 站点配置误改
- About 内容误改
- 团队 / 时间轴误操作

## 6. 稳定性与保护能力

当前系统已内置以下基础保护：

- 全站分级限流
- 管理后台更严格限流
- AI / 上传 / 配置导入并发隔离
- 公开只读接口短时缓存
- 运维中心风险提示与状态汇总

目标是：

- 防止短时大量请求拖垮单机服务
- 控制 AI、生图上传、配置导入等高成本接口堆积
- 为后续多实例、Redis、网关层限流扩展保留边界

## 8. 典型运营流程

### 8.1 内容运营

1. 管理员登录后台
2. 在作品管理中维护作品
3. 在站点配置中维护首页、About、门店和媒体资源
4. 如需新素材，可通过 AI 生图辅助生成并人工审核发布
5. 访客在前台浏览内容并提交留言
6. 管理员在后台查看留言并继续跟进

### 8.2 交付与售后

1. 使用部署脚本或离线发布包完成部署
2. 首次登录后修改管理员密码
3. 导入正式站点配置
4. 补充作品与媒体资源
5. 建立备份、巡检与升级流程

## 9. 适用场景

当前版本适合：

- 鲜花门店品牌官网
- 花艺作品展示站
- 线索收集型内容站
- 中小规模商业交付项目

当前版本不适合：

- 电商交易平台
- 高并发订单系统
- 多租户 SaaS
- 复杂角色权限系统
