# Example Prompts

Use these examples to adapt the skill to real delivery work.

## General Delivery

- "Apply the delivery-project-template skill to this repository and fix the broken admin flow."
- "Use the delivery project template and take this feature from code change through deploy."
- "Handle this as a delivery task, not just a code patch."

## Debugging And Repair

- "Use the delivery project template to trace why the deployed admin page fails after login."
- "Treat this as a data plus deployment issue and fix it end to end."
- "Use the template and repair the production-like settings state safely."

## Deployment

- "Use the delivery template and deploy the current branch to the server."
- "Run this change through the repo's official release path and verify the live URL."
- "Package this repo for customer delivery using the project template."

## Documentation

- "Use the delivery project template and update the docs to match the real deployment model."
- "Create the missing handoff docs and operations notes using the delivery template."

## High-Risk Operations

- "Use the delivery template and rotate the app secret safely."
- "Use the delivery template and fix the encrypted settings after a key mismatch."

## Multi-Client Projects

- "Use the delivery template and make sure the web app, backend, and mini app stay aligned."
- "Apply the delivery template and verify shared fields across all clients."

## Adaptation Prompt

When copying this skill into another repo, a useful first prompt is:

- "Adapt the delivery-project-template skill to this repository's stack, deployment model, and operational constraints without making it overly specific."
